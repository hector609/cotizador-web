## Resumen

- **Validación RFC**: ✅ Presente en signup (13-char regex), onboarding steps, cotizar-pdf (override form). Login confía servidor.
- **Error Display**: ✅ Todos con `role=alert`, animaciones Framer, colores rose/amber/indigo consistentes.
- **A11Y Esencial**: ✅ IDs estables `useId()`, `aria-invalid/describedby`, labels `htmlFor`, skip-to-main ausente pero forms son secondary routes.
- **Gaps**: Onboarding steps (Step1-7 components) no auditados — deben revisar labels + invalid attrs cada uno. Cotizar-pdf parcial form campos no tienen `aria-labelledby` explícito (relying on `<label>`).

**Recomendación**: Auditar `/components/onboarding/Step{1-7}.tsx` para Labels + aria-labelledby, y cotizar-pdf partial form para `aria-labelledby` en divs de grupo.
